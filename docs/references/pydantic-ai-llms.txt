## Core Concepts

Agent: Agent[DepsT, ResultT] — the core unit. ResultT is the typed output.
  - Agent(model=..., result_type=PlanOutput, system_prompt="...")
  - result = await agent.run(user_prompt) → RunResult[ResultT]
  - result.data: ResultT — the typed, validated output

Tools: @tool decorator on sync/async functions. Agents call tools automatically.
  - Return types must be serializable (Pydantic models preferred)
  - Tool docstring is the tool description shown to the model

Models: pydantic_ai.models.vertexai.VertexAIModel("gemini-2.0-flash")
  - Requires vertexai.init(project=..., location=...) first

Instrumentation: logfire.instrument_pydantic_ai() — zero-config tracing
  - Every agent run, tool call, and model response is traced as OTel spans
  - Pydantic model fields surface as structured span attributes

## Structured Output

Always use result_type= parameter, never parse text:
  agent = Agent(model=model, result_type=MyOutput, system_prompt="...")
  result = await agent.run(prompt)
  output: MyOutput = result.data  # Fully validated Pydantic model

## Error Handling

ModelRetry: raise ModelRetry("reason") inside a tool to ask the model to retry
UnexpectedModelBehavior: raised when model returns invalid structured output
